王淮：通过代码吃掉世界

发表于 2014-08-23   |   分类于 程序员大会   |  

2014 年 8 月 23 日下午「哪上班 | 程序员大会」上，线性资本创始合伙人王淮现场发表了一场名为《通过代码吃掉世界》的演讲。

以下为王淮的演讲实录：

感谢这么多人来这边听我作为一位工程师的一些体会，在开始之前我想给大家提一个问题，有多少人是工程师？举个手，好像多数都是工程师，挺好的，我本来想说的是那些不是工程师的可以花5秒钟的时间羡慕一下工程师们，因为接下来的世界是属于他们的，既然你们都是，你们可以互相羡慕！

我今天演讲的题目是《Code to eat the world》，通过代码吃掉这个世界。

技术界的耶路撒冷：硅谷

首先，我们来看一看技术界的耶路撒冷：硅谷。在硅谷聚集了上千家的大大小小的科技公司，这里面有很多大家耳熟能详的科技公司，谷歌在这里，英特尔在这个地方。硅谷是以斯坦福为中心，半径50公里的这样一个区域，这条是101从北边到南边，很难想象在这样一个小地方，出来的上千家公司改变了人类在六十年代整个的生活方式，HP跟IBM在计算机上面的一些发明创造，后来的微处理器，英特尔在这上面做出了巨大的贡献，在过去的六七年之间，移动互联网的发展，很大程度是是由硅谷的这些公司来推动的。但这里面有一个特点，我不知道大家有没有注意到，这些公司绝大多数的创始人都是工程师背景，他们都是从最一开始的头几行代码把他们的这些东西给码出来的，而这种趋势只会越来越明显。我有幸参加了当初的一家公司，Facebook。

在我一开始加入 Facebook 的时候，那时候的 Facebook 大概有 130 多人，我印象当中，那时候的用户是两千万左右，大部分是学生，收入是将近于零，有一个很小的在右边的我们叫 FLYER 的小广告，学生可以买的，5 美金，可以放上几天，所以收支在一开始从帐面上看是比较困难的。但是 Facebook 幸运的是一直以来资本都非常充分，所以钱一直不是 Facebook 的问题。时间往后，五年之后，这是 Facebook 大概今天的状态，有六千多员工，我昨天晚上看过的数据是 13 亿的月活跃用户。而最近一个季度的财报上面，它赚的钱大概是 29 亿美金，大概是60%还是40%是纯收入，很难想象在七年时间之内，Facebook 从一开始这么小的一家公司，成长到今天这么一个大的规模。这样的事情，在硅谷这个地方，在这样一个工程师驱动的一个地方，每隔几年都有一个这样传奇性的故事产生。但更多的是很多没 Facebook 那么夸张的故事在上演，很多的公司并购、上市，很多公司的产品影响到千千万万远在几千公里外的人，以前我印象非常深的是 Facebook 的一个产品可以影响到几千万人的生活，随便的一个BUG可以让几亿人哭爹喊娘。这种感觉是非常爽的，大家有没有做过产品覆盖过几千万人的？大家举个手，我相信大家可以有这样的共鸣，但是你的感觉越好，应该责任也越大。

Facebook 的经历

在 Facebook 的经历让我第一次可以感觉到可以用你的指尖让你的代码改变很多人的生活，而且是非常多的人的生活。作为一个工程师，一定是一个非常牛的工程师，但你要有这个能力去做这个事情。我待会儿会解释，大家为什么要有这能力做这个事情。我们做科技第一点是一个意愿的问题，你认为你能不能做这样的事情，这是第一个，这决定了其他的所有的剩下的做法，但有了意愿然后呢？我接下来给我们以前在整个发展过程中我的体会，还有最近两年做科技投资整个体会当中的一些建议。

成为一个产品工程师

第一点，成为一个产品工程师，而不仅仅是一个程序员工程师
我不知道大家能不能理解这里面的区别，右边画了一张图，大家看待对于解决这个问题需要的产品是什么样子的，很多不同的角色带来的对这个最终的结果的不同。作为一个产品工程师需要有两方面的技能，一方面当然是能写代码，而另一方面是站在用户的角度、产品的角度去思考。你的代码能够成什么样，在前在后，如何测试你的产品？如何获得用户的反馈？什么样的矩阵才是你想要的？很多的东西是在代码之外的东西是你要学会思考的，而这样的人在我们看到的很多的科技公司中相对比较少见。能找到有这样的产品思维的工程师都是非常有机会创业，改变这个世界的。

选择比努力更重要

我们还觉得能力是一个核心问题，什么意思？现在能够创业的，能够被互联网、移动互联网颠覆的方向非常多。但一次只能做一个，做什么方向，而这个方向是真正发自内心能够体现你雄心的方向，这是一个值得思考的问题。最近几年一个很火的话是你想要创业要站在风口上，有两类人做这个事情：一个是创业的时候自己打造一个风口，但这样的事情可能是雷军这样的大佬可以做；但更多的工程师，要思考一下，什么样一个方向，什么样一个市场中的形势让你去点评一下。八、九年前做大众点评系统是非常吃力的，但是看到点评在三四年前的飞跃很大程度上是一个技术上的成熟。所以在选择方向上，除了本身的激情以外，你的市场和方向是不是准备好了。

JUST CODE IT

第三点，作为一个工程师非常大的优势是你可以 JUST CODE IT
你不需要依托其他人做出来，你可以自己把它写出来。我在斯坦福读书的时候，经常有很多人跟我说，我有什么想法，我有什么想法。其实大家有很多好的想法，但是缺的是一个到位的执行。这个想法我有很多，我可以把它做出来，我可以看到你身上的价值，所以很多时候变成的我们要选择他们。我可以把它写出来，我可以花两周，日日夜夜，我可以把它做出来，在硅谷我可以组建我的团队，认同我的理念的这样的工程师一起做这个东西。只要我们把这个产品做出来，感觉不错，很多其他的人会自己找上门来，这是我们当时工程师创业的一个很巨大的好处，大家应该利用好这个优势。
但反过来，尤其在中国创业，工程师创业最好在越早的时候找到一个懂市场的一个合伙人，因为中国不是一个产品驱动的市场，你需要一个懂市场的人知道市场的深浅，而这样一个人是可以在中国很好的创业的。

其他前面的东西都可以通过这一步，只要你没有在失败之前被打垮，都是可以的。不断的前行，不断的学习，我们现场做一个 SP 值，玩儿游戏的人都知道这是一个经验值，在一个大公司小公司有一个两三年的锻炼，这样可能基础会好一些。但你还没有成功的时候，你要的是不断的迭代跟循环，在尝试的过程中，需要这么几项东西，你之前的经验还有你的战斗力，看看这就决定你有多大的程度可以承受整个创业过程中的压力。你做创业的产品什么样的，后面有什么样的技术支撑？假设分成一个非常好的产品，这是一个好的程序，因为成长的太快，技术跟不上，连续两周的时间经常是可上可不上，然后用户丢失了大概超过一半，接下来往这个方向走，用户体验太差了。市场选择，要知道如何把自己推广出去，中国稍微好一点，但在硅谷非常重要。如果还没有成功的话，记得一定要坐下来，给自己放个空，给自己三个月的假，好好的思考一下为什么失败了，失败在哪里，给自己增加一下生命力。下次接着再来，像韩冰刚才说的创业是一个一辈子的事情，很多创业者的一辈子都停留在成功变成1的时候，非常可怜。

软件要吃掉世界

刚才是我对工程师去通过代码改造世界的一些做法上面的建议，回到一个更高层面的问题，为什么这个馅饼去落到工程师头上而是希望人身上？

因为软件要吃掉世界，这是一个人曾写过的一篇文章，里面一个核心的思想是世界准备好了，整个的基础架构已经准备好了。像我一开始说的，花了六十年，计算机的技术到现在这个地步；花了四十年，微处理器的技术到现在这个地步；花了二十年，互联网到今天这个地步；只花了六到七年，移动互联网，人手一个智能机。绝大多数发展中国家智能机占到了很大的比例，整个的基础架构已经准备好了。我们用一个新的技术把它颠覆掉，在工程师的世界里面有这么几个特点看待世界的优势。

问题是可以被计算出来的

第一个：问题是可以被计算出来的。什么意思？我们经常把一个问题结构化，它需要一个真正的结构去处理。我们如何做迭代跟递归，很多这样思考问题的方式，非工程师是很难去理解的，而这些东西如果需要通过计算机解决大家的问题，必须要明白这些解决的理解跟解决这些问题的思考方式。而通过这种思考方式出来的答案，解决方案，利用现有的基础架构，是可以非常低的成本或者零成本的不断的被重复重复。因为你有的问题我也有，我不希望解决你的时候花的大价钱在我身上又重复一遍，但是计算机上这些东西全部自动化，以非常低的成本。

世界是平的

第二个：世界是平的。什么意思，互联网把所有人连接到一个网单中，以前大家上网的是要么是在上班，或者是在家里，但是移动互联网是随时随地，我有一个非常好的新闻，通过互联网我可以在一夜之间爆红，可以一夜之间让几千万人享受到我的产品。但是以前我要告诉 Michael，Michael 再告诉 Nathan，Nathan 再告诉其他人，这样一步一步传达过去。这个时间这个成本带来对你的产品能力是无法像今天这样发展的，等待有这样想法的工程师挖掘这样的机会。给工程师带来的挑战是如何做出更人性化的产品，而刚才更多强调的是结构化和用程序和数字表达。但是到了现代这个社会，我们非常强调的是使用体验，用户体验，我们要的是 cool 的产品，所以在这个过程中，如何把艺术、人性、设计跟你工程师创造的产品结合起来，这可能是作为工程师创业中非常大的一个挑战。

这是我们线性资本新成立的一个基金，很希望能够在这个过程中帮助大家的，我们非常希望那些老大哥们，用新的技术、方式，尤其是工程师创业来颠覆老的做事情的方式。
