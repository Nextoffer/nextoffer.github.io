AMA with DINE 创始人兼设计师杜潇

发表于 2015-11-12   |   分类于 设计   |  

嘉宾介绍：杜潇

本期嘉宾是设计师杜潇杜老师，他的网名是：disinfeqt。一位谜一样的美男子，目前是 DINE 设计工作室的创始人兼设计师，曾经在虾米音乐任职，知乎第一版的设计师。

目前 DINE 工作室的客户包括知乎，唐茶，饭本，疯果，哪上班，拇指阅读，数字尾巴等一系列国内有名的创业公司，2014 年曾获得苹果最佳应用的 GQ24 ，以及中国移动，魅族等公司。

精选问答

@香梨小金刚：杜老师结婚了吗？今天的双11光棍节打算怎么过？购物清单能给我们看看么？

哈哈，今天就吃顿饭吧。没有购物清单，想买的都买了。

@谭忐忐：杜老师一般从什么地方寻找设计灵感哇?

工作外的话我喜欢逛同行的网站；工作中需要的时候，一般是 dribbble/designspiration 吧。

@Mar§hall：同行是指各个 designagency 么，杜老师？

是。【杜老师你平时说话也是这么言简意赅吗？】

@Mar§hall：自己比较喜欢的 agency 方便列举下不？

喜欢的 agency… Metalab / Collins / Big Active。挺多的，想不起来。这三家在各自的领域比较有特色：Metalab 是传统的 UI 设计，跟我们比较类似（dinehq.com）；Collins 是品牌设计；Big Active 是创意和平面设计，设计了很多封面什么的。

@cbvivi：杜老师对今天到场的人数满意吗？

受宠若惊了都。【是的，杜老师还私聊我们问是不是请了托儿，明明是你杜老师自己的魅力吸引来的好吗！】

@kyth：杜老师能否简单介绍下如果接的是一个 app 的第一版设计的话，后续迭代时和甲方设计师的沟通模式？

是问设计交付后，甲方如何接手的问题吗？关于交付设计的问题，我们一般会提供结构良好的 styleguide 和全部设计源文件，如果可以的话我们会邀请甲方设计师参与部分细枝末节的设计，这样可以比较平滑地过渡。

@kyth：差不多吧，这里的假设是之后的版本你不再参与了，但可能之后重要新功能的设计的讨论中，会不会还有甲方“请教”你的局面出现，一般这样是怎么处理的

没什么特别的，装超人，拼命补课，然后迅速开始新的合作。「请教」类问题我们欢迎啊，哈哈。

@cbvivi：杜老师，我们甲方在跟优秀设计外包团队合作时，双方如何有尊严地有效避免如图所示的局面发生？

我们客户都很文明的…

虽然这幅图可能是杜撰的，但我觉得暴露了甲方对乙方职责不理解的问题。很多甲方认为不管是设计上还是产品上的问题，都希望丢给一个「外包公司」通过一次合作就能把自己产品积累已久的问题全部解决掉，这是不现实也不理智的。

@Tony：杜老师平时都是从什么渠道接项目的？

渠道的话，这么多年总结下来最靠谱的应该是朋友介绍和知乎。

@罗晟：杜老师今年买过的东西里最推荐的是哪一款？

Alienware 15，让我觉得 Windows 10 特别好用…

@无脸人：你会如何介绍 DINE？DINE 是一个怎样的团队？

Dine 是一支小团队。哈哈。「小团队」这个说法算是我们最大的特色了。我想大多设计公司如果被客户问起团队规模的话，小于 20 人都羞于启齿的吧。我们是理直气壮。【羡慕杜老师这种经营着小而精的团队。】

@翎心子：那杜老师的小团队目前有多少人了~？

4

@翎心子：考虑来 Nextoffer 上发告示收几个徒弟吗~？

这么多年我们没考虑过招人，很多次项目很紧真的有松动的念头，但最后还是忍住了。如果要招人 Nextoffer 肯定是首选啦，哈哈。保持小规模、最重要的是合伙人制，这样可以压榨每个人的潜能。【一个善于植入小广告的群主是可爱的。】

@论坛：求问大神，你觉得艺术或者设计这种事情，存在天赋一说吗？如果不是学设计的，后续转学设计会不会设计思维跟不上导致出来的东西没有人使用。具体事例：有朋友从小开始学习画画，各种成系统各种画画，虽然成不了艺术家但是做设计是没有问题的。如果一个从小零基础的小伙伴，坚持画画四年，除了插画师还能有别的出路吗？

UI 设计真的很简单，给会用电脑的人 PS 或 Sketch 摆弄几天都能入门。难的地方是以用户的角度去思考，设计用户爱用的产品而不是你自己爱用的产品。
